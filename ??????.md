# Stock Sentinel 新手操作指南

这份指南假设你没有编程经验。照顺序操作即可。任何需要你亲自登录、申请 Key、设置密钥或确认部署的地方，都明确写有：**需要用户手动完成**。

> 风险声明：分析结果不构成投资建议，任何指标都无法保证盈利。软件只做提醒和模拟交易，不会连接银行卡、券商或自动下单。

## 0. 先在电脑上跑通演示版

**需要用户手动完成**

1. 安装 [Python 3.11 或更高版本](https://www.python.org/downloads/)。安装时勾选 `Add Python to PATH`。
2. 安装 [Node.js 20 或更高版本](https://nodejs.org/)。
3. 在项目文件夹空白处按住 Shift 点右键，选择“在此处打开 PowerShell”。
4. 逐行运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\run-demo.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\run-frontend.ps1
```

5. 终端出现 `http://localhost:5173` 后，在浏览器打开它。
6. 页面应显示 AAPL、MSFT、NVDA，并清楚标记“演示数据”。

停止本地页面：回到 PowerShell 窗口按 `Ctrl + C`。

## 1. 如何创建 GitHub 仓库

**需要用户手动完成**

1. 打开 [GitHub](https://github.com/) 并登录，没有账户就先注册。
2. 右上角点 `+` → `New repository`。
3. Repository name 填 `stock-sentinel`。
4. 建议选择 `Private` 先测试；确认没有密钥后再改 Public。
5. 不要勾选“Add a README”“Add .gitignore”或“Choose a license”，本项目已经包含这些文件。
6. 点 `Create repository`，保留随后页面上的仓库地址。

## 2. 如何上传项目

**需要用户手动完成**

推荐安装 [GitHub Desktop](https://desktop.github.com/)：

1. 打开 GitHub Desktop，登录 GitHub。
2. 选择 `File` → `Add local repository`，选择本项目 `stock-sentinel` 文件夹。
3. 如果提示不是仓库，点 `create a repository`，名称保持 `stock-sentinel`。
4. 左下角 Summary 填 `feat: complete stock sentinel system`，点 `Commit to main`。
5. 点顶部 `Publish repository`，选择刚才的仓库并确认。

使用命令行时：

```powershell
git init -b main
git add .
git commit -m "feat: complete stock sentinel system"
git remote add origin https://github.com/你的用户名/stock-sentinel.git
git push -u origin main
```

上传前务必运行 `git status`，确认没有 `.env`。仓库中只能有 `.env.example`。

## 3. 如何申请行情 API Key

### 方案 A：yfinance（最省事）

不需要 API Key。把 `config/settings.json` 的 `provider` 改为 `yfinance`，或在 GitHub 仓库 Settings → Secrets and variables → Actions → Variables 新增：

- Name：`STOCK_DATA_PROVIDER`
- Value：`yfinance`

yfinance 不保证实时，可能延迟；仅适合个人研究，并须遵守 Yahoo 数据条款。

### 方案 B：Alpha Vantage

**需要用户手动完成**

1. 打开 [Alpha Vantage API Key 页面](https://www.alphavantage.co/support/#api-key)。
2. 按页面要求申请免费 Key。
3. 不要把 Key 发给别人，也不要写进 `settings.json`。
4. 把 Provider 设为 `alpha_vantage`，并按第 4 节添加 `ALPHA_VANTAGE_API_KEY` Secret。

免费层当前每天最多 25 次请求，免费日线通常不是实时行情。默认观察 5 只产品，每个交易日约使用 5 次请求；不要改成半小时轮询。

### 方案 C：Twelve Data

**需要用户手动完成**

1. 打开 [Twelve Data](https://twelvedata.com/) 注册并进入 API Keys 页面。
2. 复制 Key，妥善保存。
3. 把 Provider 设为 `twelve_data`，并添加 `TWELVE_DATA_API_KEY` Secret。
4. 在官方定价页确认当前免费额度和市场权限；不同套餐的新鲜度不同。

## 4. 如何设置 GitHub Secrets

**需要用户手动完成**

1. 打开 GitHub 仓库。
2. 点 `Settings` → `Secrets and variables` → `Actions`。
3. 在 `Secrets` 标签点 `New repository secret`。
4. 按你使用的服务添加，不使用的不要添加：

| Secret 名称 | 内容 |
|---|---|
| `ALPHA_VANTAGE_API_KEY` | Alpha Vantage Key |
| `TWELVE_DATA_API_KEY` | Twelve Data Key |
| `EMAIL_TO` | 收件邮箱 |
| `EMAIL_FROM` | 已验证的发件地址 |
| `RESEND_API_KEY` | Resend API Key |
| `SMTP_HOST` | SMTP 服务器 |
| `SMTP_PORT` | 通常 465 或 587 |
| `SMTP_USERNAME` | SMTP 用户名 |
| `SMTP_PASSWORD` | 邮箱授权码，不是网页登录密码 |

5. 切换到 `Variables` 标签，添加非敏感变量：

| Variable 名称 | 示例值 |
|---|---|
| `STOCK_DATA_PROVIDER` | `yfinance` |
| `EMAIL_ENABLED` | `false`，测试完成再改 `true` |
| `EMAIL_PROVIDER` | `resend` 或 `smtp` |
| `SMTP_USE_SSL` | `true` |
| `SITE_URL` | `https://用户名.github.io/stock-sentinel/` |

Secret 保存后 GitHub 不会再次完整显示，这是正常的。

## 5. 如何配置收件邮箱

### 推荐：Resend

**需要用户手动完成**

1. 打开 [Resend](https://resend.com/) 注册。
2. 创建 API Key。
3. 按 Resend 指引验证发件域名，或先使用其允许的测试发件方式。
4. 添加 `RESEND_API_KEY`、`EMAIL_TO`、`EMAIL_FROM` Secrets。
5. 添加变量 `EMAIL_PROVIDER=resend`。
6. 第一次先保持 `EMAIL_ENABLED=false`。

### 备选：SMTP

**需要用户手动完成**

1. 在邮箱安全设置中开启 SMTP。
2. 生成“授权码”或“应用专用密码”。不要使用邮箱网页登录密码。
3. 添加 `SMTP_HOST`、`SMTP_PORT`、`SMTP_USERNAME`、`SMTP_PASSWORD` Secrets。
4. 添加变量 `EMAIL_PROVIDER=smtp`。

## 6. 如何手动运行一次监测

**需要用户手动完成**

1. 打开 GitHub 仓库 → `Actions`。
2. 左侧选择“行情监测”。
3. 点 `Run workflow`。
4. `忽略交易时段限制` 保持勾选，点绿色按钮。
5. 等待任务完成。绿色勾表示成功；红色叉点进去查看日志。
6. 第一次会先做回测，再生成监测数据。仓库中的 `data/` 和 `frontend/public/data/` 会更新。

本地手动运行：

```powershell
.\.venv\Scripts\python -m stock_sentinel monitor --force
```

## 7. 如何开启 GitHub Actions 定时任务

**需要用户手动完成**

1. 在仓库 `Actions` 页确认 Actions 已启用。
2. 在 Variables 设置 `STOCK_DATA_PROVIDER=alpha_vantage`。
3. 提交 `config/settings.json` 的产品列表。
4. `monitor.yml` 已配置为工作日北京时间 16:30 触发一次；程序只在国内市场收盘分析窗口请求，以保护免费额度。

GitHub cron 可能延迟，不能用于秒级或交易所级实时监控。公共仓库长时间没有活动时，GitHub 也可能暂停计划任务，请定期检查 Actions。

`intraday.yml` 会在国内交易时段每15分钟运行一次免费盘中风险检查，无需新增 API Key。它使用 AKShare 聚合公开网页行情，页面会明确标记为实验性来源；必须执行的止损仍需在券商软件设置条件单。

## 8. 如何部署 GitHub Pages

**需要用户手动完成**

1. 仓库点 `Settings` → `Pages`。
2. 在 `Build and deployment` 的 Source 选择 `GitHub Actions`。
3. 打开 `Actions`，选择“部署 PWA 到 GitHub Pages”，点 `Run workflow`。
4. 完成后打开 `https://你的用户名.github.io/stock-sentinel/`。
5. 把这个地址填入 Variables 的 `SITE_URL`。

如果仓库是 Private，确认你的 GitHub 套餐是否允许 Private Pages。最简单的免费方案通常是确认无密钥后把仓库改为 Public。

## 10. 如何添加或删除股票

**需要用户手动完成**

方法一（适合新手）：

1. 打开 PWA → 设置 → 股票管理。
2. 添加或删除股票，填写成本、数量、止盈和止损。
3. 点“保存本机设置”。
4. 点“导出 settings.json”。
5. 在 GitHub 仓库打开 `config/settings.json` → 铅笔图标 → 用导出的内容替换 → `Commit changes`。

方法二：直接编辑 `config/settings.json`。代码只能包含字母、数字、`.` 和 `-`；市场填 `US`、`CN` 或 `HK`。A 股和港股能否获取取决于所选 Provider。

## 11. 如何修改买入和卖出阈值

**需要用户手动完成**

1. PWA → 设置 → 评分与风险阈值。
2. 修改买入和卖出阈值，范围 0–100，默认都是 65。
3. 导出并替换 GitHub 的 `config/settings.json`。
4. 手动运行一次“行情监测”，确认没有配置错误。

阈值越低提醒越多、误报也可能更多；阈值越高信号更少，但不代表准确率或收益必然更高。

## 12. 如何查看错误日志

**需要用户手动完成**

本地日志：打开 `logs/stock-sentinel.log`。

GitHub 日志：仓库 → `Actions` → 点失败任务 → 点红叉步骤。程序不会主动打印 Key，但仍不要把完整日志公开给陌生人。

常用本地检查：

```powershell
.\.venv\Scripts\python -m stock_sentinel validate
.\.venv\Scripts\python -m pytest
```

## 13. 如何关闭邮件和停止程序

**需要用户手动完成**

关闭邮件：GitHub → Settings → Secrets and variables → Actions → Variables，把 `EMAIL_ENABLED` 改为 `false`。也可以在 `config/settings.json` 中把每只股票的 `email_enabled` 改为 `false`。

停止定时监测：

1. GitHub → Actions → “行情监测”。
2. 右上角三点菜单 → `Disable workflow`。

停止本地页面：在运行 `scripts\run-frontend.ps1` 的 PowerShell 按 `Ctrl + C`。

紧急情况：先 Disable workflow，再把所有 API Key/授权码在服务商后台撤销并重新生成。
